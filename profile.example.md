# 지원자 프로필 (LLM 매칭용 요약)

> 이 파일을 `profile.md` 로 복사한 뒤 본인 이력으로 고치세요. (profile.md 는 .gitignore 됨)
> job_watcher.py 가 공고와 대조할 때 LLM 프롬프트에 넣는 이력 요약이다.
> 자유 형식이지만, 아래처럼 "한 줄 → 경력 → 기술스택 → 선호조건 → 감점요인" 순이면 LLM이 잘 읽는다.

## 한 줄
백엔드 개발자 4년차. Java/Spring 프로덕션 운영 경험. (본인 강점을 한 문장으로.)

## 경력 (N년차)
- **프로젝트 A (회사/스택)**: 무엇을 만들었고 무엇을 책임졌는지 한두 줄.
- **프로젝트 B**: ...
- **프로젝트 C**: ...

## 기술 스택
- **강점(핵심)**: Java, Spring Boot, MySQL, Redis, REST API ...
- **보조(경험 있음)**: Python, Docker, Elasticsearch ...
- **학습 중/전이 가능**: Kafka, Kubernetes ...

## 선호·조건
- 직무: 백엔드/서버 개발 (풀스택도 백엔드 비중 크면 적합).
- 경력 매칭: 3~7년 공고가 최적. 신입 전용/시니어 리드 요구는 낮음.
- 도메인 강점: (해당 도메인 나열).
- 지역: 서울/경기/원격 선호.

## 감점 요인 (솔직하게)
> LLM이 과대평가하지 않도록 부족한 부분을 정직하게 적을수록 매칭 정확도가 올라간다.
- 실무 경험 없는 기술, 미보유 자격, 부적합 직무 등을 적는다.
- 공고에서 그 항목이 **필수**면 실제 갭으로, **우대**면 경미하게 평가된다(프롬프트에 반영됨).
